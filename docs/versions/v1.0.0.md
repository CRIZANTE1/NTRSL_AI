# v1.0.0 — Shell do app e nutrição offline

**Status:** entregue  
**Escopo:** frontend React + Capacitor, sem backend de IA nem persistência na nuvem.

## Resumo

Primeira base utilizável do **NTRSL AI**: telas, navegação, autenticação (UI + Supabase Auth), registro de exercícios/alimentos e **cálculo calórico 100% no cliente** a partir de JSON embarcado.

## O que entrou

### UI e navegação

- Rotas: `/login`, `/cadastro`, `/home`, `/historico`, `/sobre`, `/profile`, `/settings/*`
- Layout com header e bottom nav
- Design System com tokens em `src/theme/colors.ts`

### Nutrição offline

- `src/data/calorias.json` e `src/data/exercicios.json`
- `src/lib/nutrition.ts` — `buildSummary()`, pickers com Fuse.js
- `MacroChart` (Recharts) na Home

### Auth (cliente)

- `AuthContext` com e-mail/senha via Supabase
- Login biométrico e bloqueio ao retornar do background (Capacitor)
- Cadastro com metadados `full_name` / `display_name`

### Infraestrutura preparada (não conectada)

- `localDb` + `outboxSync` — fila offline sem produtor
- `NetworkBanner`, `OfflineSyncEffects`
- Cliente `src/lib/api.ts` apontando para **FastAPI** (não implementado)
